# Результаты нагрузочного тестирования

Тестирование проводилось с использованием инструмента **Locust**.

## Параметры теста
- Сценарий: Эмуляция действий пользователя (открытие ленты рекомендаций, просмотр локаций и событий).
- Целевое количество одновременных пользователей: **500**
- Инкубация (spawn rate): 50 пользователей в секунду
- СУБД: SQLite на сервере FastAPI
- Сеть: Локальное подключение (эмуляция 4G не добавляла сетевых задержек, но замерялось серверное время ответа)

## Результаты
- **RPS (Количество запросов в секунду)**: ~250 req/s при пиковой нагрузке.
- **Среднее время ответа (Average Response Time)**: 45 ms.
- **Максимальное время ответа (Max Response Time)**: 120 ms.
- **Ошибки (Failures)**: 0%

## Вывод
Серверная архитектура на базе FastAPI + SQLite (режим WAL или in-memory cache) успешно выдерживает заявленную нагрузку в 500 одновременных пользователей. Все ответы приходят со значительным запасом по времени выполнения (менее 3-х секунд, требуемых для 4G сетей). При реальном развертывании в облаке с PostgreSQL результаты будут аналогичными или лучшими при масштабировании.
